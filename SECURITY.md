# Security Policy

## Supported Versions

This is a portfolio project. Security fixes are applied on a best-effort basis to the default branch.

## Reporting a Vulnerability

Please open a private report if possible. If private reporting is unavailable, open an issue without exploit details and request a secure contact channel.

When reporting, include:
- affected file/module
- reproduction steps
- impact assessment
- proposed mitigation (optional)

## Scope Notes

This project targets public website scraping and local data storage. Review your own legal/compliance obligations before running in production environments.
